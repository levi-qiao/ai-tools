## 安装与配置

### Q1: 打开文件时提示"宏已被禁用"怎么办？

**A**: 需要启用宏才能使用工具。

**解决方法**：
1. 关闭文件
2. 打开Excel选项：文件 → 选项 → 信任中心 → 信任中心设置
3. 选择"宏设置" → 勾选"启用所有宏"
4. 点击确定，重新打开文件

**安全提示**：仅对信任的文件启用宏。

### Q2: 点击"开始匹配"按钮没有反应？

**A**: 可能是宏未正确加载。

**排查步骤**：
1. 检查文件扩展名是否为 `.xlsm`（启用宏的工作簿）
2. 按 `Alt+F11` 打开VBA编辑器，检查是否有代码
3. 确认【原始数据】工作表存在且有数据
4. 尝试关闭Excel重新打开

### Q3: 可以在Mac上使用吗？

**A**: 可以，但需要注意兼容性。

**注意事项**：
- Mac版Excel的VBA功能有限制
- 部分快捷键不同（Cmd代替Ctrl）
- 建议使用Excel 2016或更高版本
- 如遇问题，建议在Windows环境使用

## 数据处理

### Q4: 为什么发票号码显示为科学计数法？

**A**: Excel默认将长数字转换为科学计数法。

**解决方法**：
1. **粘贴前设置格式**（推荐）：
   - 选中B、C、D、E、G、J列
   - 右键 → 设置单元格格式 → 文本
   - 然后粘贴数据

2. **已经粘贴的数据**：
   - 无法恢复，需要重新粘贴
   - 从源系统重新导出数据

**预防措施**：
- 工具已预设这些列为文本格式
- 每次清空数据后需重新设置

### Q5: 税收分类编码只显示前几位怎么办？

**A**: 同样是格式问题，参考Q4的解决方法。

**验证方法**：
- 正确：`1010101010000000000`（19位完整显示）
- 错误：`1.01E+18`（科学计数法）

### Q6: 可以处理多少条数据？

**A**: 理论上无限制，实际受Excel和电脑性能影响。

**性能参考**：
- 1000条：约3秒
- 5000条：约15秒
- 10000条：约30秒
- 50000条：约3分钟

**建议**：
- 单次处理不超过10000条
- 超过建议分批处理（按月份或供应商）

### Q7: 数据中有空行会影响匹配吗？

**A**: 会跳过空行，不影响结果。

**最佳实践**：
- 建议删除空行，提高处理速度
- 使用Excel的"定位空值"功能批量删除

## 规则配置

### Q8: 如何添加新的税目规则？

**A**: 在【税目映射规则】工作表添加新行。

**步骤**：
1. 找到最后一条规则
2. 在下一行填写12个字段
3. 规则编号建议按顺序（R01, R02...）
4. 优先级根据重要性设置（1-99）

**示例**：
```
R15 | 304 | 3 | 建筑服务 | 装修/施工 | (空) | 建设工程合同 | 0.3 | 征税 | 15 | (空) | (空) | 建筑装修服务
```

### Q9: 关键词匹配不区分大小写吗？

**A**: 不区分大小写。

**示例**：
- 规则关键词：`办公用品`
- 匹配：`办公用品`、`OFFICE用品`、`办公SUPPLIES`

### Q10: 可以使用正则表达式吗？

**A**: 当前版本不支持正则表达式。

**替代方案**：
- 使用多个关键词（用`/`分隔）
- 创建多条规则覆盖不同情况

### Q11: 优先级相同时如何匹配？

**A**: 按规则在表中的顺序，从上到下匹配。

**建议**：
- 避免设置相同优先级
- 特殊规则优先级设低（数字小）
- 通用规则优先级设高（数字大）

### Q12: 如何设置某个税目不征税？

**A**: 在规则的"是否征税"列填写"不征税"或"不征收"。

**示例**：
```
R99 | 6 | 1 | 不征税项目 | (空) | (空) | 不征税项目 | 0 | 不征税 | 99 | (空) | (空) | 未发生销售行为
```

## 结果验证

### Q13: 如何查看哪些发票未匹配？

**A**: 在【发票明细】工作表筛选"匹配税目"列。

**步骤**：
1. 点击M列（匹配税目）的筛选按钮
2. 勾选"未匹配"
3. 查看筛选结果

**处理方法**：
- 检查税收分类编码是否正确
- 在规则表中添加对应规则
- 重新运行匹配

### Q14: "是否排除"为"是"的发票需要处理吗？

**A**: 需要人工确认是否正确。

**常见排除原因**：
- 不征税项目（编码6开头）
- 红字发票（负数金额）
- 规则设置为"不征收"

**验证方法**：
- 查看"排除原因"列（R列）
- 确认是否符合业务实际
- 如有疑问，咨询财务人员

### Q15: 汇总金额与预期不符怎么办？

**A**: 逐步排查问题。

**排查步骤**：
1. 检查【发票明细】中的"应纳税额"列
2. 使用Excel的SUM函数手动计算
3. 对比【季度汇总】的公式
4. 检查是否有重复数据

**常见原因**：
- 税率配置错误
- 数据重复粘贴
- 公式引用错误

## 性能问题

### Q16: 运行时Excel卡死怎么办？

**A**: 实际上是在处理，不是卡死。

**判断方法**：
- 查看状态栏（底部）是否显示进度百分比
- 如显示进度，说明正在运行，请耐心等待

**如果真的卡死**：
1. 等待5分钟
2. 仍无响应则按 `Ctrl+Break` 中断
3. 保存文件，重启Excel
4. 减少数据量后重试

### Q17: 如何提高处理速度？

**A**: 采用以下优化措施。

**数据层面**：
- 删除不必要的列和空行
- 分批处理大量数据
- 关闭其他Excel文件

**系统层面**：
- 关闭其他占用内存的程序
- 确保硬盘空间充足
- 使用较新版本的Excel

**工具层面**：
- 当前版本已优化（使用数组批量处理）
- 1000条数据约3秒完成

### Q18: 可以在后台运行吗？

**A**: 不建议，可能导致错误。

**原因**：
- VBA需要操作工作表
- 切换窗口可能中断执行
- 建议运行时不要操作Excel

## 错误排查

### Q19: 提示"运行时错误'9': 下标越界"？

**A**: 工作表名称或结构不匹配。

**检查项**：
1. 确认存在以下工作表：
   - 原始数据
   - 发票明细
   - 税目映射规则
   - 季度汇总
2. 工作表名称完全一致（不能有空格差异）
3. 【原始数据】至少有3行（标题+示例+数据）

### Q20: 提示"运行时错误'1004': 应用程序定义或对象定义错误"？

**A**: 单元格操作失败。

**常见原因**：
- 工作表被保护
- 单元格被锁定
- 公式引用错误

**解决方法**：
1. 检查工作表是否被保护（审阅 → 撤销工作表保护）
2. 确认没有合并单元格
3. 检查公式是否正确

### Q21: 匹配结果全部为"未匹配"？

**A**: 规则表可能为空或格式错误。

**检查项**：
1. 打开【税目映射规则】工作表
2. 确认至少有一条规则（第3行开始）
3. 检查"编码前缀"列是否有值
4. 检查"位数"列是否为数字

### Q22: 负数发票没有被排除？

**A**: 检查规则的"负数发票处理"列。

**逻辑说明**：
- 第12列为空或"不征税" → 负数发票不征税
- 第12列为"征税" → 负数发票也征税

**验证方法**：
1. 找到对应规则
2. 查看第12列的值
3. 根据业务需求调整

### Q23: 六税两费减免没有生效？

**A**: 检查配置和公式。

**检查项**：
1. 【季度汇总】工作表E2单元格是否填写"是"
2. F列（减免税额）公式是否正确：`=IF($E$2="是",D5*0.5,0)`
3. G列（实际应纳税额）公式是否正确：`=D5-F5`

**注意**：
- 必须填写"是"（不是"√"或"Y"）
- 公式引用必须是 `$E$2`（绝对引用）

## 其他问题

### Q24: 可以导出结果到其他格式吗？

**A**: 可以，使用Excel的导出功能。

**支持格式**：
- CSV（逗号分隔值）
- PDF（打印预览）
- TXT（制表符分隔）

**步骤**：
1. 选择要导出的工作表
2. 文件 → 另存为
3. 选择文件类型
4. 保存

### Q25: 如何备份规则配置？

**A**: 复制【税目映射规则】工作表。

**方法1：工作表复制**
1. 右键点击【税目映射规则】标签
2. 选择"移动或复制"
3. 勾选"建立副本"
4. 重命名为"税目映射规则_备份_20260428"

**方法2：文件备份**
1. 另存为新文件
2. 文件名添加日期后缀
3. 保存到备份文件夹

### Q26: 工具会自动更新吗？

**A**: 不会自动更新，需要手动下载新版本。

**获取更新**：
1. 访问GitHub仓库
2. 查看Releases页面
3. 下载最新版本
4. 手动替换文件

**迁移数据**：
- 复制【税目映射规则】工作表到新版本
- 重新粘贴【原始数据】
