## 基础操作

### 第一步：准备数据

从税务系统导出发票数据，确保包含以下字段：

| 必需字段 | 说明 | 示例 |
|---------|------|------|
| 发票号码 | 8位数字 | 12345678 |
| 销方识别号 | 18位统一社会信用代码 | 91110000XXXXXXXXXX |
| 税收分类编码 | 19位编码 | 1010101010000000000 |
| 货物名称 | 商品或服务名称 | 办公用品*打印纸 |
| 价税合计 | 含税金额 | 1130.00 |

### 第二步：设置列格式

**重要**：粘贴数据前必须设置文本格式，否则长数字会变成科学计数法。

1. 打开【原始数据】工作表
2. 选中以下列（按住Ctrl多选）：
   - B列（发票代码）
   - C列（发票号码）
   - D列（数电发票号码）
   - E列（销方识别号）
   - G列（购方识别号）
   - J列（税收分类编码）
3. 右键 → 设置单元格格式 → 文本 → 确定

### 第三步：粘贴数据

1. 从第3行开始粘贴（第1行是标题，第2行是示例）
2. 确保列对齐正确
3. 检查税收分类编码是否完整显示（19位）

### 第四步：运行匹配

1. 点击工具栏的"开始匹配"按钮
2. 等待进度提示（状态栏显示百分比）
3. 完成后自动跳转到【发票明细】工作表

### 第五步：检查结果

在【发票明细】工作表中：

| 列 | 字段 | 说明 |
|----|------|------|
| M | 匹配税目 | 自动匹配的税目名称 |
| N | 应纳税额 | 计算的印花税金额 |
| O | 征税状态 | 应税/不征收 |
| P | 匹配规则 | 使用的规则编号 |
| Q | 是否排除 | 是/否 |
| R | 排除原因 | 不征税/红字发票等 |

**重点关注**：
- 匹配税目为"未匹配"的行 → 需人工判断
- 是否排除为"是"的行 → 确认是否正确

### 第六步：查看汇总

切换到【季度汇总】工作表：

1. 查看各税目的汇总金额
2. 设置"是否符合六税两费"（E2单元格）
3. 查看实际应纳税额

## 规则配置

### 规则结构

每条规则包含12个字段，按优先级从高到低匹配。

### 配置示例

#### 示例1：买卖合同（货物销售）

```
规则编号: R01
编码前缀: 101
位数: 3
编码含义: 货物
关键词判断: (留空，所有货物都匹配)
排除关键词: (留空)
对应税目: 买卖合同
税率: 0.3‰
是否征税: 征税
优先级: 1
负数发票处理: (留空)
备注: 货物销售类发票
```

**匹配逻辑**：税收分类编码前3位 = `101`

#### 示例2：运输合同（货运服务）

```
规则编号: R03
编码前缀: 3010506
位数: 7
编码含义: 运输服务
关键词判断: 货运/运费
排除关键词: 客运/票价
对应税目: 运输合同
税率: 0.3‰
是否征税: 征税
优先级: 3
负数发票处理: (留空)
备注: 货运服务，排除客运
```

**匹配逻辑**：
1. 编码前7位 = `3010506`
2. 货物名称包含"货运"或"运费"
3. 货物名称不包含"客运"或"票价"

#### 示例3：不征税项目

```
规则编号: R99
编码前缀: 6
位数: 1
编码含义: 未发生销售行为的不征税项目
关键词判断: (留空)
排除关键词: (留空)
对应税目: 不征税项目
税率: 0
是否征税: 不征税
优先级: 99
负数发票处理: (留空)
备注: 不征税项目
```

### 优先级规则

- 数字越小，优先级越高
- 建议范围：1-99
- 特殊规则（如不征税）设置为99

### 关键词语法

**包含关键词**（第5列）：
- 单个关键词：`办公用品`
- 多个关键词（或关系）：`货运/运费/物流`
- 含义：货物名称包含任意一个关键词即匹配

**排除关键词**（第10列）：
- 单个关键词：`客运`
- 多个关键词（或关系）：`客运/票价/旅客`
- 含义：货物名称包含任意一个关键词则不匹配

## 高级功能

### 负数发票处理

**业务背景**：
- 红字发票用于冲销原发票
- 会计准则：冲销不重复征税
- 默认行为：负数发票不征税

**特殊场景**：
某些业务需要对负数发票征税，在规则的第12列填写"征税"。

**示例**：
```
规则编号: R03
...
负数发票处理: 征税
```

此规则匹配的负数发票也会征税。

### 六税两费减免

**政策依据**：
- 财政部 税务总局公告2023年第12号
- 适用对象：小型微利企业、个体工商户
- 减免比例：50%
- 适用期限：2023.1.1 - 2027.12.31

**操作步骤**：
1. 打开【季度汇总】工作表
2. 找到第2行"是否符合六税两费"
3. 在E2单元格填写：
   - `是` → 享受50%减免
   - `否` → 不享受减免
4. 查看"实际应纳税额"列（自动计算）

**计算公式**：
```
实际应纳税额 = 应纳税额 × (1 - 50%)
```

### 批量修改规则

**场景**：需要调整多个规则的税率

**方法**：
1. 打开【税目映射规则】工作表
2. 选中需要修改的单元格区域
3. 使用Excel的查找替换功能（Ctrl+H）
4. 批量替换税率值

**示例**：
- 查找：`0.3`
- 替换为：`0.5`
- 点击"全部替换"

## 最佳实践

### 数据准备

1. **定期备份**：每次处理前备份原始数据
2. **数据清洗**：删除空行和无效数据
3. **格式统一**：确保日期、金额格式一致

### 规则维护

1. **定期审查**：每季度检查规则准确性
2. **版本控制**：重大修改前保存副本
3. **文档记录**：在备注列说明规则用途

### 结果验证

1. **抽样检查**：随机抽取10%数据人工核对
2. **异常排查**：重点检查"未匹配"和"排除"的数据
3. **金额核对**：汇总金额与预期对比

### 性能优化

1. **分批处理**：超过5000条数据建议分批
2. **关闭自动保存**：处理时暂时关闭自动保存
3. **清理历史数据**：定期清空【原始数据】和【发票明细】

### 常见错误

| 错误 | 原因 | 解决方法 |
|------|------|---------|
| 科学计数法 | 未设置文本格式 | 粘贴前设置列格式为文本 |
| 大量未匹配 | 规则不完整 | 补充对应编码的规则 |
| 金额异常 | 税率配置错误 | 检查规则表税率列 |
| 运行卡死 | 数据量过大 | 分批处理或等待进度完成 |

### 技巧提示

1. **快速定位**：使用Ctrl+F查找特定发票号码
2. **筛选功能**：使用Excel筛选功能查看特定税目
3. **数据透视**：使用数据透视表进行多维分析
4. **公式审计**：使用Excel公式审计功能检查计算逻辑
