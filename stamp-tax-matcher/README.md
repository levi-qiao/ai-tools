# 印花税自动匹配工具

## 📋 简介

基于Excel VBA的印花税自动匹配系统，根据发票的税收分类编码和货物名称自动匹配对应的印花税税目，支持六税两费减免计算。

## ✨ 核心特性

- ✅ 自动匹配印花税税目（买卖合同、运输合同、租赁合同等）
- ✅ 支持六税两费50%减免
- ✅ 自动处理红字发票（负数发票）
- ✅ 科学计数法自动修复（19位税收分类编码）
- ✅ 季度汇总报表自动生成
- ✅ 性能优化（批量处理，10倍速度提升）
- ✅ 所有规则在Excel中配置，无需修改代码

## 🚀 快速开始

### 1. 打开工具

双击打开 `印花税自动匹配模型.xlsm`

### 2. 粘贴发票数据

1. 切换到【原始数据】工作表
2. 从第3行开始粘贴发票数据
3. 确保包含以下列：
   - 发票号码
   - 销方识别号
   - 税收分类编码
   - 货物名称
   - 价税合计

### 3. 运行匹配

点击"开始匹配"按钮，系统会自动：
- 匹配印花税税目
- 计算应纳税额
- 标记需人工确认的项目

### 4. 查看结果

- **发票明细**：查看每张发票的匹配结果
- **季度汇总**：查看按税目汇总的数据

### 5. 六税两费减免

在【季度汇总】工作表：
1. 找到第2行"是否符合六税两费"
2. 在E2单元格填写：
   - `是` - 享受50%减免
   - `否` - 不享受减免
3. 查看"实际应纳税额"列

## 🔧 配置规则

### 税目映射规则表

在【税目映射规则】工作表中配置匹配规则：

| 列 | 字段名 | 说明 |
|----|--------|------|
| A | 规则编号 | 唯一标识（如R01） |
| B | 编码前缀 | 税收分类编码前缀 |
| C | 位数 | 匹配前几位 |
| D | 编码含义 | 说明文字 |
| E | 关键词判断 | 货物名称必须包含的关键词（用/分隔） |
| F | 对应税目 | 匹配成功后的税目 |
| G | 税率(‰) | 千分之几 |
| H | 是否征税 | 征税/不征收 |
| I | 优先级 | 数字越小越优先 |
| J | 排除关键词 | 包含则不匹配（用/分隔） |
| K | 备注说明 | 规则说明 |
| L | 负数发票处理 | 填"征税"则负数发票也征税，留空则不征税 |

### 规则示例

```
规则编号: R03
编码前缀: 3010506
位数: 7
关键词判断: 货运/运费
排除关键词: 客运/票价
对应税目: 运输合同
税率: 0.3‰
是否征税: 征税
优先级: 3
负数发票处理: (留空，红字发票不征税)
```

**匹配逻辑**：
1. 税收分类编码前7位 = `3010506`
2. 货物名称包含"货运"或"运费"
3. 货物名称不包含"客运"或"票价"
4. 正数发票征税，负数发票不征税

## ⚠️ 注意事项

### 数据粘贴前

**重要**：粘贴数据前，先将以下列设置为文本格式：
- 第2列：发票代码
- 第3列：发票号码
- 第4列：数电发票号码
- 第5列：销方识别号
- 第7列：购方识别号
- 第10列：税收分类编码

**操作方法**：
1. 选中这些列（按住Ctrl多选）
2. 右键 → 设置单元格格式 → 文本
3. 然后粘贴数据

**原因**：防止长数字（如19位税收分类编码）被Excel自动转换为科学计数法。

### 负数发票处理

- **默认行为**：负数发票（红字发票）不征税，标记为"红字发票"
- **特殊需求**：如需对某规则的负数发票征税，在该规则的第12列填写"征税"

### 六税两费减免

**适用对象**：
- 小型微利企业
- 个体工商户

**减免比例**：50%

**适用期限**：2023年1月1日至2027年12月31日

## 🐛 常见问题

### Q1：税收分类编码显示为科学计数法

**A**：粘贴数据前，先将第10列设置为文本格式。详见 [FAQ Q4](./docs/faq.md#q4-为什么发票号码显示为科学计数法)

### Q2：大量发票显示"未匹配"

**A**：检查【税目映射规则】中是否有对应的编码前缀规则。详见 [FAQ Q13](./docs/faq.md#q13-如何查看哪些发票未匹配)

### Q3：负数发票是否征税？

**A**：默认不征税（红字冲销）。如需征税，在规则的第12列填写"征税"。详见 [FAQ Q22](./docs/faq.md#q22-负数发票没有被排除)

### Q4：如何添加新的匹配规则？

**A**：在【税目映射规则】工作表最后一行添加新规则，填写完整信息。详见 [使用指南 - 规则配置](./docs/guide.md#规则配置)
